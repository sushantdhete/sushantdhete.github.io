
**Step 3 — Enable GitHub Pages**
```
Repository → Settings → Pages
Source: Deploy from a branch

```

**Step 4 — Access Your Site**
```

(May take 1–5 minutes to propagate)
```


## 🔧 Customization Guide

### 1. Replace Your Name
Search and replace `Sushant Ashok Dhete` in all HTML files.

### 2. Add Your Photo
Replace the avatar placeholder in `index.html`:
```html
<!-- Find this: -->
<div class="hero-avatar-placeholder">SAD</div>

<!-- Replace with: -->
<img src="assets/images/your-photo.jpg" alt="Sushant Ashok Dhete" style="width:100%;height:100%;object-fit:cover;border-radius:50%"/>
```

### 3. Update Contact Details
In `contact.html` and `index.html`, update:
- Email address
- LinkedIn URL
- GitHub URL
- WhatsApp number

### 4. Add Real Projects
In `js/projects.js`, edit the `PROJECTS` array:
```javascript
{
  id: 1,
  title: 'Your Project Name',
  category: 'ai', // 'ai' | 'web' | 'automation'
  tags: ['React', 'Python'],
  icon: '🤖',
  color: '#5b6ef5',
  desc: 'Short description...',
  challenge: 'What was hard...',
  solution: 'How you solved it...',
  results: 'Measurable outcomes...',
  tech: ['Tech1', 'Tech2'],
  demo: 'https://your-demo.com',
  github: 'https://github.com/you/repo',
}
```

### 5. Add Real Certificates
Replace placeholder images in `assets/certificates/` and update `data-cert` attributes in `certifications.html`.

### 6. Custom Domain (Optional)
Create a `CNAME` file in root:
```
yourdomain.com
```

## 🎤 Viva Q&A

**Q: Why no frameworks like React or Vue?**
> "The project requirements specified vanilla HTML/CSS/JS for GitHub Pages compatibility. This also demonstrates deeper understanding of core web fundamentals — anyone can use a framework, but knowing the DOM API, CSS animations, and Canvas directly shows mastery."

**Q: How does the dark/light theme work?**
> "I use a `data-theme` attribute on the `<html>` element combined with CSS custom properties. Toggling the attribute switches the entire color system instantly. The preference is saved to `localStorage` so it persists across sessions."

**Q: Explain the particle animation.**
> "I use the HTML5 Canvas API. Each particle is an object with x/y position and velocity. On every animation frame via `requestAnimationFrame`, I clear the canvas and redraw all particles at their new positions. Lines are drawn between particles within 120px of each other, creating the connected network effect."

**Q: How does the intersection observer work for scroll animations?**
> "The `IntersectionObserver` API watches DOM elements and fires a callback when they enter or leave the viewport. I add a `.reveal` class to elements I want to animate, then add `.visible` when they scroll into view, which triggers the CSS transition."

**Q: How is the skill bar animation triggered?**
> "Skill bars start at `width: 0%`. An IntersectionObserver watches each bar. When it scrolls into the viewport, I set `width` to the target percentage via a `data-pct` attribute. CSS `transition: width 1.2s cubic-bezier(...)` handles the smooth animation."

**Q: What accessibility features did you implement?**
> "ARIA labels on all interactive elements, semantic HTML5 landmarks (nav, main, section, footer), keyboard navigation support, proper color contrast ratios, screen-reader-compatible form validation with `role='alert'`, and a prefers-reduced-motion media query that disables animations for users who prefer it."

**Q: How does the contact form validation work?**
> "Each field has a validator function that returns an error message or empty string. On blur, the field is validated. On submit, all fields are validated and if any fail, submission is blocked. Error messages are displayed in adjacent `<span>` elements with `role='alert'` for screen readers."

**Q: Why did you choose this color palette?**
> "The purple-to-teal gradient (indigo #5b6ef5 + teal #00d4aa) creates a modern, tech-forward aesthetic seen in tools like Linear and Vercel. Deep navy backgrounds reduce eye strain. The system uses CSS variables throughout so it's trivial to theme."

**Q: How does the AI chat widget work?**
> "It's a demo UI built with CSS positioning and JavaScript. The chat interface toggles open/closed with transform and opacity transitions. User messages are appended to the DOM, and after 800ms a pre-written bot response is shown. In a production version, this would call an actual LLM API."

**Q: How would you add a real backend to this?**
> "For the contact form, I'd add a FastAPI endpoint or use a service like Formspree/EmailJS. For the AI widget, I'd connect to the Anthropic or OpenAI API. For analytics, I'd add a lightweight tracking pixel or use Google Analytics."
